إنشاء الـ Organizational Units (OUs)

OUs تنظّم المستخدمين والمجموعات — مثل مجلدات داخل Active Directory

١
افتح Active Directory Users and Computers من Server Manager ← Tools
٢
كليك يمين على company.local ← New ← Organizational Unit
٣
اسمها: company ← اضغط OK
٤
افتح الـ OU الجديدة وأنشئ جوّها:
company.local
  └── 📁 company (OU)
      ├── 📁 Users (OU للمستخدمين)
      │     ├── 👤 HR (sub-OU)
      │     ├── 👤 Finance (sub-OU)
      │     └── 👤 IT (sub-OU)
      └── 📁 Groups (OU للمجموعات)
👥
إنشاء الـ Security Groups
١
كليك يمين على OU الـ Groups ← New ← Group
٢
Group name: HR_Group ← Group scope: Global ← Group type: Security ← OK
٣
كرر لإنشاء: Finance_Group و IT_Group
اسم المجموعةGroup ScopeGroup Typeالغرض
HR_GroupGlobalSecurityموظفو قسم HR
Finance_GroupGlobalSecurityموظفو قسم Finance
IT_GroupGlobalSecurityموظفو قسم IT
👤
إنشاء المستخدمين
١
كليك يمين على OU المناسبة مثل Users/HR ← New ← User
٢
أدخل الاسم الأول والأخير وـ User logon name مثل: ahmad.ali@company.local
٣
اضغط Next ← ضع كلمة مرور ← شيك على Password never expires (في بيئة تجريبية) ← Next ← Finish
٤
أضف المستخدم لمجموعته: كليك يمين عليه ← Add to a group ← اكتب HR_Group ← OK

إعداد الصلاحيات الكاملة
Share Permissions · NTFS · ABE
🔗
Share Permissions — المجلد الرئيسي Shared
١
كليك يمين على C:\Shared ← Properties
٢
تبويب Sharing ← Advanced Sharing
٣
✅ شيك على Share this folder ← Share name: Shared
٤
اضغط Permissions ← احذف Everyone ❌
٥
اضغط Add ← اضغط Locations ← اختر company.local ← اكتب Authenticated Users ← Check Names ← OK
٦
اعطِها Read فقط ← OK
🏠
NTFS — المجلد الرئيسي Shared
١
تبويب Security ← Advanced
٢
اضغط Disable Inheritance ← اختر Convert inherited permissions into explicit permissions
٣
احذف كل الـ entries ما عدا SYSTEM و Administrators
٤
اضغط Add ← Select a principal ← اكتب COMPANY\Authenticated Users ← OK
٥
في Applies to اختر This folder only — ثم شيك: Read & Execute + List folder contents + Read
٦
اضغط OK ← Apply ← OK
📂
NTFS — المجلدات الفرعية (HR / Finance / IT)

كرر على كل مجلد فرعي — مثال على HR:

١
كليك يمين على HR ← Properties ← Security ← Advanced
٢
اضغط Disable Inheritance ← Remove all inherited permissions from this object
٣
أضف SYSTEM: Full Control ← This folder, subfolders and files
٤
أضف Administrators: Full Control ← This folder, subfolders and files
٥
أضف COMPANY\HR_Group: Read & Execute ← This folder, subfolders and files
٦
اضغط OK ← Apply ← OK
⚠️لا تضيف Authenticated Users على المجلدات الفرعية أبداً — ABE سيتولى الإخفاء

الخيار الأول: Synology NAS

NAS = Network Attached Storage — جهاز تخزين يتصل بالشبكة ويدعم LDAP/Active Directory

الخطوة ١ — إعداد Synology NAS
١
ادخل على واجهة NAS عبر المتصفح: http://[IP-NAS]:5000
٢
أكمل معالج الإعداد الأولي واختر اسم للـ NAS
٣
عيّن IP ثابت للـ NAS مثل: 192.168.1.40
الخطوة ٢ — ربط NAS بـ Active Directory
١
في Synology DSM افتح Control Panel ← Domain/LDAP
٢
اختر Domain ← اكتب: company.local
٣
أدخل بيانات Admin الدومين ← اضغط Apply
٤
الآن يمكنك إضافة مستخدمي الدومين على الـ NAS مباشرة
الخطوة ٣ — إنشاء Shared Folders على NAS
١
افتح Control Panel ← Shared Folder ← Create
٢
اسم المجلد: HR ← اضغط Next
٣
في Permissions أضف COMPANY\HR_Group ← اعطيها Read/Write
٤
كرر لـ Finance و IT
الخطوة ٤ — ربط NAS بالـ Clients عبر GPO
١
على DC01 افتح Group Policy Management ← أنشئ GPO جديد: Map-NAS-HR
٢
اضغط Edit ← User Configuration ← Preferences ← Windows Settings ← Drive Maps
٣
كليك يمين ← New ← Mapped Drive
٤
Location: \\192.168.1.40\HR ← Drive Letter: H: ← Label: HR Drive
٥
في تبويب Common شيك على Item-level targeting ← Targeting ← أضف شرط: Security Group = HR_Group
٦
اربط الـ GPO بـ company.local
✅الآن موظف HR سيجد Drive H: مرتبط تلقائياً بمجلد HR على الـ NAS بمجرد تسجيل الدخول

الخيار الثاني: Azure Files

خدمة Microsoft السحابية — يمكن ربطها مع Active Directory On-Premises

الخطوة ١ — إنشاء Storage Account على Azure
١
ادخل على portal.azure.com وسجّل دخول
٢
ابحث عن Storage accounts ← Create
٣
اختر Subscription والـ Resource Group ← اعطِ اسماً: companystorage
٤
Region: اختر الأقرب ← Performance: Standard ← Review + Create
الخطوة ٢ — إنشاء File Share
١
افتح الـ Storage Account ← من القائمة اليسرى اختر File shares
٢
اضغط + File share ← اسمه: hr-files ← Tier: Transaction optimized
٣
كرر لـ finance-files و it-files
الخطوة ٣ — ربط Azure Files بـ AD On-Premises
PowerShell — DC01 
# ثبّت AzFilesHybrid module
Install-Module -Name AzFilesHybrid -Force

# سجّل دخول لـ Azure
Connect-AzAccount

# فعّل AD Authentication
Join-AzStorageAccountForAuth `
  -ResourceGroupName "your-resource-group" `
  -StorageAccountName "companystorage" `
  -DomainAccountType "ComputerAccount" `
  -OrganizationalUnitDistinguishedName "OU=Computers,DC=company,DC=local"
الخطوة ٤ — ربط Drive عبر GPO
١
في Azure Portal اضغط على الـ File Share ← Connect
٢
اختر Active Directory ← انسخ سكريبت PowerShell اللي يظهر
٣
أو استخدم UNC Path في GPO Drive Maps: \\companystorage.file.core.windows.net\hr-files
ℹ️Azure Files يوفر إمكانية الوصول من أي مكان في العالم مع نفس صلاحيات الـ AD

الأسهل والأكثر انتشاراً في الشركات الصغيرة والمتوسطة
